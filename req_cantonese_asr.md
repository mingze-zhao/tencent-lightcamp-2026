# 家访秒记 — 粤语转录需求文档

## 1. 概述

**模块名称**：粤语转录（ASR：语音转文字）  
**目标**：将社工家访时录制的粤语对话（可中英夹杂）转成完整、可读的文本，供后续 Agent 做结构化抽取与报告生成。

**在整体架构中的位置**：前端上传音频 → **本模块**接收音频并返回转写文本 → Agent 模块使用转写文本进行抽取与报告。

---

## 2. 功能需求

### 2.1 输入要求

| 项目 | 说明 |
|------|------|
| **音频来源** | 由前端或后端上传的录音文件；格式需与所选 ASR 服务要求一致（常见：WAV、MP3、M4A、WebM 等）。 |
| **时长** | 单段建议支持 1–30 分钟（Demo 可先支持 1–5 分钟）；超长音频可考虑分片或后端先切分再逐段转写。 |
| **内容特点** | 粤语为主；可能夹杂英文（如药名 Metformin）、少量普通话；口语化、拉家常式对话。 |
| **声道** | 单声道或双声道均可，以 ASR 服务支持为准；若仅单声道，需在录音或上传前约定。 |

### 2.2 输出要求

| 项目 | 说明 |
|------|------|
| **主输出** | 完整转写文本（`transcript`），与录音内容对应，可用于直接展示在「左侧转写」区域。 |
| **格式** | 纯文本或带时间戳的 JSON（如 `segments: [{ start, end, text }]`）；P0 至少保证有连续完整文本。 |
| **说话人** | P0 可选：若 ASR 支持说话人分离（Diarization），可输出「谁在什么时候说了什么」，便于后续扩展；若不支持，仅输出整段转写即可。 |
| **语言** | 转写结果以繁体中文呈现为佳（与香港社福场景一致）；若服务仅输出简体，可在后处理做简繁转换。 |

### 2.3 质量与性能

| 项目 | 说明 |
|------|------|
| **准确率** | 粤语识别准确率尽量高；对常见药名、身体部位（如脚肿、头晕）等关键词需可识别。 |
| **延迟** | Demo 允许同步转写（整段上传后一次性返回）；单次 3–5 分钟音频，建议 30 秒内返回结果。 |
| **鲁棒性** | 对轻微背景噪音、口音有一定容错；失败时返回明确错误信息供前端展示。 |

---

## 3. 接口规范（建议）

### 3.1 转写接口（由后端对外暴露）

**方式一：上传即转写（推荐 Demo 使用）**

- **Request**：`POST /api/transcribe`（或 `/api/upload`，由后端统一命名）  
- **Body**：`multipart/form-data`，字段名如 `file`，值为音频文件。  
- **Response 成功**：

```json
{
  "text": "完整转写文本，可直接用于左侧展示与后续抽取",
  "segments": [
    { "start": 0.0, "end": 2.5, "text": "阿婆今日点啊？" }
  ]
}
```

- **Response 失败**：HTTP 4xx/5xx，body 中给出 `error` 或 `message`。

**方式二：先上传后转写**

- 上传返回 `audio_id` 或 URL，再调用 `POST /api/transcribe` 传入 `audio_id`/URL，返回同上。  
- P0 可采用方式一简化实现。

## 4. 技术选型建议

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **腾讯云 ASR（粤语）** | 粤语支持好、与比赛方生态一致、文档完善 | 需注册与配置 | 首选，尤其面向香港/腾讯比赛 |
| **Whisper API 或自建 Whisper** | 支持粤语（language=yue）、效果较好 | API 需网络；自建需算力 | 备选或快速验证 |
| **阿里云/讯飞等** | 有粤语引擎可选 | 需单独对接 | 按团队已有资源选择 |

**P0 建议**：优先对接一家云 ASR（如腾讯云），实现「上传音频 → 返回 text」的最小闭环；说话人分离若有现成能力可顺带接入，否则 P0 可不做。

---

## 5. 实现方式（与后端的关系）

- **独立服务**：粤语转录可实现在后端同一项目中，作为单独路由或子模块；接收音频 → 调用所选 ASR SDK/API → 将返回结果整理为上述 JSON 返回给前端或 Agent 调用方。  
- **无状态**：单次请求进、单次结果出；不需要持久化音频（Demo 可不在服务器落盘，仅做流式或临时文件传入 ASR）。  
- **安全与合规**：Demo 使用假数据或脱敏录音；若涉及真实个案，需在后端做访问控制与审计，并遵守隐私政策。

---

## 6. 验收标准（P0 Demo）

- [ ] 给定一段 1–3 分钟粤语录音（可预录），调用转写接口能返回完整、可读的转写文本。  
- [ ] 转写文本可被前端展示在左侧，且能作为 `/api/extract` 的输入被 Agent 正确解析。  
- [ ] 接口超时或 ASR 报错时，能返回明确错误信息，前端可提示用户重试或检查格式。  
- [ ] 与前端联调：前端上传录音后，能在 1 分钟内（视长度而定）拿到转写并展示。

---

## 7. 依赖与对接

- **上游**：前端或后端上传的音频文件；需约定格式（如 WAV 16kHz 单声道）以符合部分 ASR 要求。  
- **下游**：转写文本提供给前端展示及 Agent 的 `/api/extract` 使用。  
- **环境**：需配置所选 ASR 服务的 API Key / Secret 等，建议通过环境变量注入，不写死在代码中。
