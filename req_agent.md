# 家访秒记 — Agent 开发需求文档

## 1. 概述

**模块名称**：Agent 开发（后端 LLM 抽取与报告生成）  
**目标**：接收粤语家访转写文本，通过大模型完成「结构化信息抽取」与「一键生成书面报告」，并输出可前端展示的 JSON 与报告正文。

**在整体架构中的位置**：前端上传音频 → 粤语转录模块输出转写 → **本模块**接收转写，输出结构化数据 + 报告 → 前端双屏展示。

---

## 2. 功能需求

### 2.1 结构化信息抽取（/api/extract）

| 项目 | 说明 |
|------|------|
| **输入** | 家访对话的完整转写文本（粤语口语为主，可能中英夹杂） |
| **输出** | 符合约定 schema 的 JSON，包含各维度摘要与风险等级 |
| **调用方** | 前端或后端流水线，在转写完成后调用 |

**抽取维度与字段约定**：

| 维度 | 字段名 | 说明 | 风险等级取值 |
|------|--------|------|--------------|
| 用药依从性 | `medication` | 是否按时服药、漏服频次、对药物的疑问等 | high / medium / low |
| 身体症状 | `symptoms` | 如头晕、脚肿、气喘、疼痛等，需保留原文关键描述 | 每项可带 risk |
| 饮食情况 | `diet` | 食欲、饮食种类、是否偏食等 | high / medium / low |
| 情绪/精神状态 | `emotion` | 情绪低落、焦虑、睡眠等 | high / medium / low |
| 日常活动能力 | `adl` | 行动、如厕、落街等能力变化 | high / medium / low |
| 社交/支援网络 | `social_support` | 家人探访、独居、照顾者情况等 | high / medium / low |
| 待跟进事项 | `action_items` | 社工或长者提出的待办（如「问医生换药」） | - |
| 预警列表 | `warnings` | 需优先关注的短句，如「连续两天未服降压药」 | - |

**风险等级与前端展示**：

- `high` → 🔴 红色预警  
- `medium` → 🟡 黄色注意  
- `low` → 🟢 绿色/稳定  

**非功能要求**：

- 输出必须为合法 JSON，便于前端直接解析；若 LLM 返回非 JSON，需有兜底解析或重试策略。
- 响应时间建议控制在单次请求 10–30 秒内（视模型与文本长度而定）。

---

### 2.2 一键生成报告（/api/report）

| 项目 | 说明 |
|------|------|
| **输入** | 上述结构化抽取得到的 JSON（或主要字段） |
| **输出** | 符合香港社福机构常用格式的书面家访报告正文（繁体中文，可选中英双语） |
| **调用方** | 前端在拿到抽取结果后调用，或后端在 extract 之后自动调用 |

**报告内容要求**：

- 语体：书面语，将口语表述转化为正式用语（如「食咗药未」→「个案表示已按时服用药物」）。
- 结构建议：个案基本信息（可占位）、本次家访日期、观察与访谈摘要（按维度分段）、风险评估与预警、建议跟进事项。
- 长度：单次家访报告建议 300–800 字，便于社工快速浏览与存档。

**非功能要求**：

- 输出纯文本或简单 Markdown，前端可原样展示或提供复制/下载。

## 3. 接口规范（建议）

### 3.1 POST /api/extract

**Request**

```json
{
  "transcript": "（完整转写文本）"
}
```

**Response（示例）**

```json
{
  "medication": { "summary": "个案表示有时忘记食降压药", "risk": "high" },
  "symptoms": [{ "description": "对脚成日肿", "risk": "medium" }],
  "diet": { "summary": "食唔落嘢，成日饮粥", "risk": "medium" },
  "emotion": { "summary": "老伴走咗之后成日眼湿湿", "risk": "high" },
  "adl": { "summary": "落街要扶住墙", "risk": "medium" },
  "social_support": { "summary": "个仔好耐冇返嚟", "risk": "low" },
  "action_items": ["联系主诊医生讨论换药可能"],
  "warnings": ["连续两天未服降压药"]
}
```

### 3.2 POST /api/report

**Request**

```json
{
  "extract": { /* 上述抽取结果 JSON */ }
}
```

**Response**

```json
{
  "report": "（完整报告正文）"
}
```

## 4. 技术要点

- **技术栈建议**：Python + FastAPI；或 Node + Express。与粤语转录模块同机或同项目内部署即可。
- **LLM 调用**：腾讯混元 / OpenAI / 通义 / 豆包等任选；需两次调用：① 抽取（transcript → JSON）② 报告（JSON → report）。
- **Prompt 设计**：
  - 抽取 Prompt：明确列出上述维度与字段、粤语例句、输出 JSON 格式及 risk 取值说明；要求「仅输出 JSON，不要多余说明」。
  - 报告 Prompt：输入为 JSON 摘要，输出为书面报告；可提供 1–2 段社福报告范例以对齐风格。
- **错误与重试**：网络超时、模型限流时应有重试或友好错误信息返回前端。

---

## 5. 验收标准（P0 Demo）

- [ ] 给定一段粤语家访转写，调用 `/api/extract` 可返回结构完整、字段齐全的 JSON。  
- [ ] 抽取结果中至少包含：用药、症状、情绪、action_items、warnings，且 risk 与 warnings 可供前端标红/黄。  
- [ ] 给定抽取 JSON，调用 `/api/report` 可返回一段连贯、书面的家访报告。  
- [ ] 与前端联调：前端能上传/选择转写 → 请求 extract → 请求 report → 右侧展示卡片与报告无报错。

---

## 6. 依赖与对接

- **上游**：依赖「粤语转录」模块输出的 `transcript` 文本。  
- **下游**：前端根据本模块返回的 JSON 与 report 做双屏展示（左侧转写，右侧结构化卡片 + 预警 + 报告）。  
- **环境**：需配置所选 LLM 的 API Key 或访问凭证，建议通过环境变量注入。
